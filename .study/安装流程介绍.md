# Hermes Agent 安装流程介绍

本文档详细介绍了 Hermes Agent 的安装脚本 (`install.sh`) 的完整安装流程。

## 📋 安装脚本概览

Hermes Agent 安装脚本是一个功能完整的自动化安装程序，支持 Linux、macOS 和 Android/Termux 平台。

### 基本用法

```bash
# 标准安装
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
# 带选项安装
curl -fsSL ... | bash -s -- --no-venv --skip-setup
```

### 可用选项

- `--no-venv` - 不创建虚拟环境
- `--skip-setup` - 跳过交互式设置向导
- `--branch NAME` - 指定要安装的 Git 分支（默认：main）
- `--dir PATH` - 指定安装目录（默认：~/.hermes/hermes-agent）
- `-h, --help` - 显示帮助信息

## � curl 命令使用详解

### 语法结构

```bash
curl -fsSL [URL] | bash -s -- [选项]
```

**参数说明：**
- `-f` - 失败时静默退出
- `-s` - 静默模式，不显示进度条
- `-S` - 显示错误信息
- `-L` - 跟随重定向
- `-s --` - 将后续参数传递给脚本

### 📌 常用安装命令

#### 1. 标准安装（推荐）

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

**特点：**
- 创建虚拟环境
- 运行交互式设置向导
- 安装所有依赖和可选组件

---

#### 2. 跳过虚拟环境

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --no-venv
```

**特点：**
- 不创建虚拟环境
- 直接使用系统 Python
- 适用于已有 Python 环境或希望全局安装的用户

**注意事项：**
- 可能影响其他 Python 项目的依赖隔离
- 需要系统已安装 Python 3.11+

---

#### 3. 跳过设置向导

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --skip-setup
```

**特点：**
- 安装完成后不运行设置向导
- 适用于自动化部署或 CI/CD
- 可在安装后手动运行 `hermes setup` 配置

---

#### 4. 指定 Git 分支

```bash
# 安装开发分支
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --branch dev

# 安装特定版本分支
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --branch v1.0.0
```

**特点：**
- 安装非 main 分支的代码
- 适用于测试新功能或使用特定版本
- 开发者和测试人员常用

---

#### 5. 自定义安装目录

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --dir /custom/path
```

**特点：**
- 安装到指定目录
- 适用于多用户共享环境
- 可配合 `HERMES_INSTALL_DIR` 环境变量使用

---

#### 6. 组合选项

```bash

# 自定义分支 + 自定义目录
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash -s -- --branch dev --dir ~/hermes-dev
```

### 📊 选项对比表

| 命令 | 虚拟环境 | 设置向导 | 适用场景 |
|------|---------|---------|---------|
| `curl ... \| bash` | ✅ 创建 | ✅ 运行 | 普通用户安装 |
| `curl ... \| bash -s -- --no-venv` | ❌ 不创建 | ✅ 运行 | 开发者/已有环境 |
| `curl ... \| bash -s -- --skip-setup` | ✅ 创建 | ❌ 跳过 | 自动化部署 |
| `curl ... \| bash -s -- --no-venv --skip-setup` | ❌ 不创建 | ❌ 跳过 | CI/CD/容器 |
| `curl ... \| bash -s -- --branch dev` | ✅ 创建 | ✅ 运行 | 测试新功能 |

### ⚠️ 注意事项

1. **`--no-venv` 限制**
   - 需要系统已安装 Python 3.11+
   - 可能与其他 Python 项目冲突
   - 不推荐在共享服务器上使用

2. **`--skip-setup` 后续步骤**
   - 安装后必须运行 `hermes setup` 配置
   - 否则无法正常使用 Hermes Agent

3. **`--branch` 风险**
   - 开发分支可能不稳定
   - 生产环境建议使用 main 分支

4. **网络要求**
   - 需要访问 GitHub 和 astral.sh
   - 某些选项可能需要额外的外部依赖

5. **权限说明**
   - 安装脚本本身**不需要 sudo**
   - 只有安装可选系统包时才需要 sudo
   - 可以拒绝安装系统包，功能仍可使用

### 🔍 故障排查

#### 问题：`bash: -s: invalid option`

**原因：** 缺少 `--` 分隔符

**解决：**
```bash
# 错误
curl ... | bash -s --no-venv

# 正确
curl ... | bash -s -- --no-venv
```

#### 问题：`curl: command not found`

**解决：**
```bash
# Ubuntu/Debian
sudo apt install curl

# CentOS/RHEL
sudo yum install curl

# macOS
brew install curl
```

#### 问题：`Permission denied`

**原因：** 脚本执行权限

**解决：**
```bash
# 方式1：使用 bash 执行
curl ... | bash

# 方式2：先下载再执行
curl -fsSL ... -o install.sh
chmod +x install.sh
./install.sh
```

## �🔄 安装流程详解

### 1️⃣ 初始化阶段

#### 1.1 配置和变量设置
- 设置安装路径：`~/.hermes/hermes-agent`
- 设置 Python 版本：3.11
- 设置 Node.js 版本：22 LTS
- 配置 Git 仓库 URL（支持 SSH 和 HTTPS）

#### 1.2 检测交互模式
- 检测是否在交互式终端中运行
- 确保脚本在 `curl | bash` 模式下也能正常工作

#### 1.3 解析命令行参数
- 解析用户提供的安装选项
- 验证参数合法性

---

### 2️⃣ 系统检测阶段

#### 2.1 操作系统检测
脚本会自动检测以下平台：
- **Linux** - 检测具体发行版（Ubuntu、Debian、Fedora、Arch 等）
- **macOS** - Apple macOS 系统
- **Android/Termux** - Termux 环境
- **Windows** - 显示错误信息，提示使用 PowerShell 安装器

#### 2.2 检测结果输出
```
✓ Detected: linux (ubuntu)
```

---

### 3️⃣ 依赖检查和安装阶段

#### 3.1 Git 检查
- 检查 Git 是否已安装
- 如果未安装，自动安装 Git
- Termux 使用 `pkg install git`
- Linux 发行版使用各自的包管理器
- macOS 使用 `xcode-select --install` 或 `brew install git`

#### 3.2 uv 包管理器检查
- 检查 uv 是否已安装（常见位置：`~/.local/bin/uv`、`~/.cargo/bin/uv`）
- 如果未安装，自动从 astral.sh 安装 uv
- uv 是快速的 Python 包管理器

#### 3.3 Python 版本检查
- 检查 Python 3.11 是否可用
- Termux 使用 `pkg install python`
- 其他平台使用 uv 安装 Python 3.11
- **无需 sudo 权限**

#### 3.4 Node.js 检查
- 检查 Node.js 22 LTS 是否已安装
- 如果未安装，自动下载并安装到 `~/.hermes/node/`
- 创建符号链接到 `~/.local/bin/`
- 支持多种架构（x64、arm64、armv7l）

#### 3.5 系统依赖检查
检查并安装以下系统级依赖：

**ripgrep** - 快速文件搜索工具
- Termux: `pkg install ripgrep`
- Ubuntu/Debian: `sudo apt install ripgrep`
- Fedora: `sudo dnf install ripgrep`
- Arch: `sudo pacman -S ripgrep`
- macOS: `brew install ripgrep`

**ffmpeg** - TTS 语音消息处理
- Termux: `pkg install ffmpeg`
- Ubuntu/Debian: `sudo apt install ffmpeg`
- Fedora: `sudo dnf install ffmpeg`
- Arch: `sudo pacman -S ffmpeg`
- macOS: `brew install ffmpeg`

**Termux 特定依赖**：
- clang、rust、make、pkg-config、libffi、openssl

#### 3.6 构建工具检查（Debian/Ubuntu）
- 检查 build-essential、python3-dev、libffi-dev
- 如果缺失，提示用户安装

---

### 4️⃣ 代码获取阶段

#### 4.1 克隆仓库
- 尝试使用 SSH 克隆（支持私有仓库）
- 如果 SSH 失败，自动回退到 HTTPS
- 如果目录已存在，执行 git pull 更新

#### 4.2 本地更改处理
- 检测并暂存本地更改
- 更新后可选择恢复本地更改
- 确保更新过程安全

---

### 5️⃣ 虚拟环境设置阶段

#### 5.1 创建虚拟环境
- Termux: 使用 Python stdlib venv
- 其他平台: 使用 uv venv
- 指定 Python 版本：3.11

#### 5.2 隔离安装
- 所有 Python 依赖安装在虚拟环境中
- 不影响系统 Python 环境
- **无需 sudo 权限**

---

### 6️⃣ Python 依赖安装阶段

#### 6.1 升级基础工具
```bash
pip install --upgrade pip setuptools wheel
```

#### 6.2 安装主包
- Termux: `pip install -e '.[termux]' -c constraints-termux.txt`
- 其他平台: `uv pip install -e '.[all]'`
- 如果 [all] 失败，回退到基础安装

#### 6.3 可选组件
- **tinker-atropos** - RL 训练组件（默认不安装）
- 可通过 `uv pip install -e "./tinker-atropos"` 后续安装

---

### 7️⃣ Node.js 依赖安装阶段

#### 7.1 前端依赖
- 安装 npm 依赖（浏览器工具）
- 安装 Playwright Chromium 浏览器引擎
- 安装浏览器系统依赖（需要 sudo）

#### 7.2 WhatsApp 桥接
- 安装 WhatsApp bridge 依赖
- 用于 WhatsApp 消息平台支持

---

### 8️⃣ 命令路径配置阶段

#### 8.1 创建符号链接
- Termux: `ln -sf $PREFIX/bin/hermes`
- 其他平台: `ln -sf ~/.local/bin/hermes`

#### 8.2 更新 PATH
- 检测用户登录 shell（bash、zsh 等）
- 更新 shell 配置文件：
  - `~/.bashrc` / `~/.bash_profile`
  - `~/.zshrc` / `~/.zprofile`
  - `~/.profile`（Ubuntu/Debian/WSL）
- 添加 `~/.local/bin` 到 PATH

#### 8.3 立即生效
- 导出 PATH 环境变量
- 使 hermes 命令在当前会话中可用

---

### 9️⃣ 配置文件设置阶段

#### 9.1 创建目录结构
```
~/.hermes/
├── cron/           # 定时任务
├── sessions/       # 会话存储
├── logs/           # 日志文件
├── pairing/        # 配对信息
├── hooks/          # 钩子脚本
├── image_cache/    # 图片缓存
├── audio_cache/    # 音频缓存
├── memories/       # 记忆存储
├── skills/         # 技能插件
├── whatsapp/
│   └── session/    # WhatsApp 会话
├── .env            # 环境变量
├── config.yaml     # 配置文件
└── SOUL.md         # 人格定义
```

#### 9.2 配置文件模板
- 从 `cli-config.yaml.example` 复制配置文件
- 创建 `.env` 文件（从 `.env.example`）
- 创建 `SOUL.md` 用于自定义代理人格

#### 9.3 同步内置技能
- 运行 `skills_sync.py` 同步技能
- 将内置技能复制到 `~/.hermes/skills/`

---

### 🔟 交互式设置向导阶段

#### 10.1 运行 setup 命令
- 调用 `hermes setup` 交互式向导
- 从 `/dev/tty` 读取用户输入
- 即使脚本通过 `curl | bash` 运行也能正常工作

#### 10.2 配置内容
- 选择默认模型提供商
- 配置 API 密钥
- 设置终端后端（local、docker、ssh 等）
- 配置消息平台令牌（可选）

#### 10.3 跳过选项
- 使用 `--skip-setup` 选项可跳过此阶段
- 可在安装后运行 `hermes setup` 手动配置

---

### 1️⃣1️⃣ 消息网关启动阶段

#### 11.1 检测配置
- 检查 `.env` 文件中是否有消息平台令牌
- 检测 Telegram、Discord、Slack、WhatsApp 等

#### 11.2 WhatsApp 配对
- 如果启用 WhatsApp 且无会话，提示用户配对
- 运行 `hermes whatsapp` 扫描 QR 码
- 非交互模式下跳过

#### 11.3 网关安装
- 提示安装系统服务（Linux systemd / macOS launchd）
- 可在安装后手动运行 `hermes gateway install`

---

### 1️⃣2️⃣ 完成阶段

#### 12.1 成功提示
```
✓ Hermes Agent installed successfully!
✓ hermes command ready
```

#### 12.2 下一步
- 运行 `hermes` 开始使用
- 运行 `hermes setup` 配置（如果跳过）
- 运行 `hermes gateway` 启动消息网关

- **自动回退** - SSH 失败自动使用 HTTPS

## ⚠️ 常见问题

### 1. 权限问题
- 安装脚本本身**不需要 sudo**
- 只有安装可选系统包时才需要 sudo
- 可以拒绝安装系统包，功能仍可使用

### 2. 网络问题
- 如果下载失败，检查网络连接
- 可以手动下载后重新运行安装

### 3. 路径问题
- 安装完成后可能需要重新打开终端
- 或运行 `source ~/.bashrc` 刷新环境

### 4. Termux 特殊情况
- Termux 使用 Python stdlib venv 而非 uv
- 浏览器和 WhatsApp 工具默认不安装
- 需要额外配置才能使用
