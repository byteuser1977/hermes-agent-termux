## 2. 核心组件职责

### 2.1 AIAgent 类 (`run_agent.py`)
- **职责**：核心对话循环控制器
- **关键方法**：
  - `chat(message)` - 简单接口，返回最终响应字符串
  - `run_conversation(...)` - 完整接口，返回包含最终响应和消息历史的字典
- **循环逻辑**：
  ```python
  while api_call_count < max_iterations and iteration_budget.remaining > 0:
      response = client.chat.completions.create(...)
      if response.tool_calls:
          for tool_call in response.tool_calls:
              result = handle_function_call(tool_call.name, tool_call.args, task_id)
              messages.append(tool_result_message(result))
      else:
          return response.content
  ```

### 2.2 工具系统 (`model_tools.py`, `tools/`)
- **职责**：工具的发现、注册、编排和执行
- **关键函数**：
  - `_discover_tools()` - 从工具集中发现可用工具
  - `handle_function_call()` - 根据工具名称分发到具体实现
  - `registry.register()` - 工具注册接口
- **工具分类**：
  - 终端工具：`terminal_tool.py`
  - 网络工具：`web_tools.py`, `browser_tool.py`
  - 文件工具：`file_tools.py`
  - 代码执行：`code_execution_tool.py`
  - 子代理：`delegate_tool.py`
  - MCP 集成：`mcp_tool.py`
  - 环境后端：`environments/` (local, docker, ssh, modal, daytona, singularity)

### 2.3 CLI 系统 (`cli.py`, `hermes_cli/`)
- **职责**：交互式命令行界面
- **关键组件**：
  - `HermesCLI` 类 - CLI 主控制器
  - `process_command()` - 命令分发
  - 皮肤引擎 (`hermes_cli/skin_engine.py`) - 主题定制
  - 命令注册表 (`hermes_cli/commands.py`) - 所有斜杠命令定义

### 2.4 消息网关 (`gateway/`)
- **职责**：多平台消息适配
- **支持平台**：Telegram, Discord, Slack, WhatsApp, Signal, Email, Home Assistant 等
- **关键模块**：
  - `run.py` - 主循环和消息分发
  - `session.py` - 会话存储
  - `platforms/` - 各平台适配器

### 2.5 状态管理 (`hermes_state.py`)
- **职责**：会话历史和记忆的持久化
- **实现**：SQLite + FTS5 全文搜索
- **功能**：
  - 会话 CRUD
  - 跨会话搜索
  - LLM 摘要生成

## 3. 数据流

```
用户输入 (CLI/Gateway)
    ↓
AIAgent.chat() / run_conversation()
    ↓
构建消息历史 (system/user/assistant/tool)
    ↓
调用 LLM API (client.chat.completions.create)
    ↓
检查 tool_calls
    ├─ 有工具调用 → handle_function_call() → 执行工具 → 添加 tool_result → 继续循环
    └─ 无工具调用 → 返回最终响应
    ↓
格式化输出 (CLI: Rich/prompt_toolkit | Gateway: 平台适配器)
    ↓
持久化到 SessionDB
```

## 4. 配置系统

- **用户配置**：`~/.hermes/config.yaml`
- **环境变量**：`~/.hermes/.env`
- **配置加载**：`hermes_cli/config.py` 中的 `DEFAULT_CONFIG` 和 `OPTIONAL_ENV_VARS`
- **Profile 支持**：通过 `HERMES_HOME` 环境变量实现多实例隔离

## 5. 扩展机制

### 5.1 添加新工具
1. 创建 `tools/your_tool.py`
2. 实现工具函数并调用 `registry.register()`
3. 在 `toolsets.py` 中添加工具集定义
4. 更新 `model_tools.py` 中的工具发现逻辑（如需要）

### 5.2 添加斜杠命令
1. 在 `hermes_cli/commands.py` 的 `COMMAND_REGISTRY` 中添加 `CommandDef`
2. 在 `cli.py` 的 `process_command()` 中添加处理逻辑
3. 在 `gateway/run.py` 中添加网关处理（如需要）

### 5.3 添加皮肤主题
- **用户皮肤**：创建 `~/.hermes/skins/<name>.yaml`
- **内置皮肤**：在 `hermes_cli/skin_engine.py` 的 `_BUILTIN_SKINS` 字典中添加

## 6. 安全与隔离

- **命令审批**：`tools/approval.py` 检测危险命令
- **容器隔离**：支持 Docker、Singularity 等后端
- **DM 配对**：网关中的用户验证机制
- **Profile 隔离**：每个 profile 有独立的 `HERMES_HOME` 目录

## 7. 依赖关系

```
tools/registry.py (无依赖，被所有工具文件导入)
       ↑
tools/*.py (每个文件在导入时调用 registry.register())
       ↑
model_tools.py (导入工具注册表并触发工具发现)
       ↑
run_agent.py, cli.py, batch_runner.py, environments/
```

## 8. 测试架构

- **测试框架**：Pytest (~3000 个测试)
- **测试隔离**：`tests/conftest.py` 中的 `_isolate_hermes_home` fixture
- **Profile 测试**：mock `Path.home()` 和 `HERMES_HOME` 到临时目录
