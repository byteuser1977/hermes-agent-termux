## 2. AIAgent 模块 (`run_agent.py`)

### 2.1 模块职责
- 实现核心对话循环
- 管理消息历史和迭代预算
- 协调工具调用和 LLM 响应

### 2.2 类设计

```python
class AIAgent:
    """
    AI 代理核心类，实现工具增强的对话循环。
    
    属性:
        model: 当前使用的 LLM 模型标识符
        max_iterations: 最大 API 调用次数
        enabled_toolsets: 启用的工具集列表
        disabled_toolsets: 禁用的工具集列表
        quiet_mode: 是否启用静默模式
        save_trajectories: 是否保存轨迹用于训练
        platform: 运行平台 (cli, telegram, discord, etc.)
        session_id: 会话 ID
        iteration_budget: 迭代预算跟踪器
    
    方法:
        chat(message) -> str: 简单对话接口
        run_conversation(...) -> dict: 完整对话接口
        _build_initial_messages(...): 构建初始消息历史
        _should_continue(): 检查是否继续循环
    """
```

### 2.3 关键流程

```
初始化
    ↓
构建消息历史 (system + user + context_files + memory)
    ↓
进入循环
    ├─ 调用 LLM API
    ├─ 检查 tool_calls
    │   ├─ 有调用 → 执行工具 → 添加结果 → 继续
    │   └─ 无调用 → 返回响应
    ├─ 检查迭代预算
    └─ 达到上限 → 终止
    ↓
保存轨迹 (可选)
```

### 2.4 依赖关系
- `model_tools.py`: 工具发现和分发
- `agent/prompt_builder.py`: 系统提示构建
- `hermes_state.py`: 会话持久化
- `agent/display.py`: 输出格式化

## 3. 工具编排模块 (`model_tools.py`)

### 3.1 模块职责
- 发现可用工具
- 将工具调用分发到具体实现
- 处理工具执行结果

### 3.2 核心函数

#### `_discover_tools(enabled_toolsets, disabled_toolsets)`
```python
"""
发现可用工具

参数:
    enabled_toolsets: 启用的工具集列表
    disabled_toolsets: 禁用的工具集列表

返回:
    list[dict]: 工具 schema 列表

流程:
    1. 导入所有 tools/*.py 模块
    2. 从 registry 获取所有工具
    3. 根据 toolsets 过滤
    4. 检查每个工具的 check_requirements()
    5. 返回可用的工具 schema
"""
```

#### `handle_function_call(tool_name, tool_args, task_id)`
```python
"""
处理工具调用

参数:
    tool_name: 工具名称
    tool_args: 工具参数 (dict 或 JSON 字符串)
    task_id: 任务 ID (用于跟踪)

返回:
    str: JSON 格式的执行结果

流程:
    1. 从 registry 获取工具 handler
    2. 解析参数
    3. 执行工具函数
    4. 捕获异常并格式化
    5. 返回 JSON 结果
"""
```

### 3.3 工具注册装饰器

### 7.3 平台适配器接口

```python
class BasePlatform(ABC):
    """平台适配器基类"""
    
    @abstractmethod
    async def send_message(self, chat_id: str, text: str):
        pass
    
    @abstractmethod
    async def send_file(self, chat_id: str, file_path: str):
        pass
    
    @abstractmethod
    def get_event(self) -> asyncio.Queue:
        pass
```

### 7.5 事件处理流程

```
用户发送消息到 Telegram
    ↓
TelegramPlatform 接收更新
    ↓
解析为 GatewayEvent
    ↓
放入 event_queue
    ↓
GatewayRunner 获取事件
    ↓
_dispatch_event(event)
    ↓

    ↓

    ↓

```

```python
def _init_db(self):
    """初始化数据库和 FTS5 索引"""
    with sqlite3.connect(self.db_path) as conn:
        # 创建 sessions 表
        conn.execute("""
            CREATE TABLE IF NOT EXISTS sessions (...)
        """)
        
        # 创建 messages 表
        conn.execute("""
            CREATE TABLE IF NOT EXISTS messages (...)
        """)
        
        # 创建 FTS5 虚拟表
        conn.execute("""
            CREATE VIRTUAL TABLE IF NOT EXISTS messages_fts 
            USING fts5(content, content_rowid=id)
        """)
        
        # 创建触发器（自动同步）
        conn.execute("""
            CREATE TRIGGER messages_ai AFTER INSERT ON messages ...
        """)
```

### 13.2 Cron 表达式格式

```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期 (0-6, 0=周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
```

示例：
- `0 9 * * *` - 每天 9:00
- `*/5 * * * *` - 每 5 分钟

## 16. 模块扩展指南

### 16.1 添加新工具

1. 创建 `tools/your_tool.py`
2. 实现工具函数
3. 调用 `registry.register()`
4. 在 `toolsets.py` 中添加工具集

### 16.2 添加斜杠命令

1. 在 `hermes_cli/commands.py` 添加 `CommandDef`
2. 在 `cli.py` 添加处理逻辑
3. 在 `gateway/run.py` 添加网关处理（可选）

### 16.3 添加皮肤主题

1. 创建 `~/.hermes/skins/<name>.yaml`
2. 定义颜色、spinner、branding
3. 使用 `/skin <name>` 激活

### 16.4 添加平台适配器

1. 创建 `gateway/platforms/your_platform.py`
2. 继承 `BasePlatform`
3. 实现 `send_message`, `send_file`, `get_event`
4. 在 `gateway/run.py` 注册
