# 数据库设计

Hermes Agent 使用 SQLite 作为主要的数据持久化方案，通过 FTS5（Full-Text Search 5）实现高效的全文搜索功能。

## 1. 数据库概览

| 数据库文件 | 路径 | 用途 |
|-----------|------|------|
| `sessions.db` | `~/.hermes/sessions.db` | 会话历史和消息存储 |
| `memory.db` | `~/.hermes/memory.db` | 长期记忆存储（可选插件） |
| `config.yaml` | `~/.hermes/config.yaml` | 用户配置（YAML 格式） |
| `.env` | `~/.hermes/.env` | API 密钥和环境变量 |

## 2. sessions.db 数据库设计

### 2.1 sessions 表

存储会话元数据。

```sql
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,              -- 会话 ID (UUID)
    platform TEXT NOT NULL,           -- 平台标识 (cli, telegram, discord, etc.)
    created_at INTEGER NOT NULL,      -- 创建时间戳 (Unix timestamp)
    updated_at INTEGER NOT NULL,      -- 最后更新时间戳
    metadata TEXT,                    -- JSON 格式的元数据
    user_profile TEXT,                -- 用户画像摘要 (可选)
    context_summary TEXT              -- 上下文摘要 (用于压缩)
);
```

**字段说明**：
- `id`: 会话唯一标识符，格式为 UUID v4
- `platform`: 来源平台，用于区分 CLI 和不同消息平台
- `created_at` / `updated_at`: Unix 时间戳（秒）
- `metadata`: JSON 字符串，包含：
  - `model`: 使用的模型
  - `toolsets`: 启用的工具集
  - `skin`: UI 皮肤
  - `personality`: 人格设定
- `user_profile`: 从对话中提取的用户画像摘要
- `context_summary`: 对话历史压缩摘要

**索引**：
```sql
CREATE INDEX idx_sessions_platform ON sessions(platform);
CREATE INDEX idx_sessions_updated_at ON sessions(updated_at DESC);
```

### 2.2 messages 表

存储会话中的消息历史。

```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    role TEXT NOT NULL CHECK(role IN ('system', 'user', 'assistant', 'tool')),
    content TEXT NOT NULL,
    timestamp INTEGER NOT NULL,
    tool_call_id TEXT,                -- 工具调用 ID (仅 tool 角色)
    tool_name TEXT,                   -- 工具名称 (仅 tool 角色)
    token_count INTEGER,              -- 估算的 token 数量
    reasoning TEXT,                   -- 推理内容 (仅 assistant 角色)
    FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);
```

**字段说明**：
- `id`: 自增主键
- `session_id`: 外键，关联到 sessions 表
- `role`: 消息角色，遵循 OpenAI API 规范
- `content`: 消息内容（纯文本或 JSON）
- `timestamp`: 消息创建时间戳
- `tool_call_id`: 工具调用 ID，用于关联 assistant 的 tool_calls 和 tool 结果
- `tool_name`: 执行的工具名称
- `token_count`: 估算的 token 数量，用于用量统计
- `reasoning`: 模型的推理过程（如使用 reasoning 模型）

**索引**：
```sql
CREATE INDEX idx_messages_session_id ON messages(session_id);
CREATE INDEX idx_messages_role ON messages(role);
CREATE INDEX idx_messages_timestamp ON messages(timestamp DESC);
CREATE INDEX idx_messages_tool_name ON messages(tool_name) WHERE tool_name IS NOT NULL;
```

### 2.3 messages_fts 虚拟表（FTS5）

用于全文搜索的虚拟表。

```sql
CREATE VIRTUAL TABLE messages_fts USING fts5(
    content,                    -- 消息内容
    content_rowid=id            -- 关联到 messages.id
);
```

**触发器**（自动同步 messages 和 messages_fts）：

```sql
-- 插入触发器
CREATE TRIGGER messages_ai AFTER INSERT ON messages BEGIN
    INSERT INTO messages_fts(rowid, content) VALUES (new.id, new.content);
END;

-- 删除触发器
CREATE TRIGGER messages_ad AFTER DELETE ON messages BEGIN
    INSERT INTO messages_fts(messages_fts, rowid, content) VALUES('delete', old.id, old.content);
END;

-- 更新触发器
CREATE TRIGGER messages_au AFTER UPDATE ON messages BEGIN
    INSERT INTO messages_fts(messages_fts, rowid, content) VALUES('delete', old.id, old.content);
    INSERT INTO messages_fts(rowid, content) VALUES (new.id, new.content);
END;
```

**搜索示例**：

```sql
-- 搜索包含关键词的消息
SELECT m.* 
FROM messages_fts f 
JOIN messages m ON f.rowid = m.id 
WHERE f.content MATCH 'Python';

-- 在指定会话中搜索
SELECT m.* 
FROM messages_fts f 
JOIN messages m ON f.rowid = m.id 
WHERE f.content MATCH 'API key' 
  AND m.session_id = 'abc123';

-- 模糊搜索（前缀匹配）
SELECT m.* 
FROM messages_fts f 
JOIN messages m ON f.rowid = m.id 
WHERE f.content MATCH 'pyth*';

-- 多词搜索
SELECT m.* 
FROM messages_fts f 
JOIN messages m ON f.rowid = m.id 
WHERE f.content MATCH 'database AND schema';
```

### 2.4 tool_executions 表

记录工具执行的详细信息，用于审计和用量分析。

```sql
CREATE TABLE tool_executions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    message_id INTEGER,
    tool_name TEXT NOT NULL,
    toolset TEXT,                     -- 工具集名称
    arguments TEXT,                   -- JSON 格式参数
    result TEXT,                      -- 执行结果
    success INTEGER NOT NULL,         -- 1=成功，0=失败
    error_message TEXT,               -- 错误信息
    execution_time_ms INTEGER,        -- 执行耗时（毫秒）
    timestamp INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id),
    FOREIGN KEY (message_id) REFERENCES messages(id)
);
```

**索引**：
```sql
CREATE INDEX idx_tool_executions_session ON tool_executions(session_id);
CREATE INDEX idx_tool_executions_tool_name ON tool_executions(tool_name);
CREATE INDEX idx_tool_executions_timestamp ON tool_executions(timestamp DESC);
```

### 2.5 usage_stats 表

存储用量统计信息。

```sql
CREATE TABLE usage_stats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,               -- 日期 (YYYY-MM-DD)
    model TEXT NOT NULL,
    platform TEXT,
    total_tokens INTEGER NOT NULL,    -- 总 token 数
    input_tokens INTEGER,             -- 输入 token 数
    output_tokens INTEGER,            -- 输出 token 数
    api_calls INTEGER NOT NULL,       -- API 调用次数
    tool_calls INTEGER,               -- 工具调用次数
    total_cost_usd REAL,              -- 估算成本（美元）
    UNIQUE(date, model, platform)
);
```

**统计查询示例**：

```sql
-- 今日用量
SELECT SUM(total_tokens) as tokens, SUM(total_cost_usd) as cost
FROM usage_stats
WHERE date = date('now');

-- 按模型统计本周用量
SELECT model, SUM(total_tokens) as tokens, SUM(total_cost_usd) as cost
FROM usage_stats
WHERE date >= date('now', '-7 days')
GROUP BY model;

-- 过去 30 天趋势
SELECT date, SUM(total_tokens) as tokens
FROM usage_stats
WHERE date >= date('now', '-30 days')
GROUP BY date
ORDER BY date;
```

## 5. 数据库操作流程

### 5.1 创建会话

```python
def create_session(platform, session_id=None):
    session_id = session_id or str(uuid.uuid4())
    conn = sqlite3.connect(DB_PATH)
    conn.execute(
        "INSERT INTO sessions (id, platform, created_at, updated_at) VALUES (?, ?, ?, ?)",
        (session_id, platform, int(time.time()), int(time.time()))
    )
    conn.commit()
    conn.close()
    return session_id
```

### 5.2 添加消息

```python
def add_message(session_id, role, content, tool_call_id=None, tool_name=None):
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.cursor()
    
    cursor.execute(
        "INSERT INTO messages (session_id, role, content, timestamp, tool_call_id, tool_name) VALUES (?, ?, ?, ?, ?, ?)",
        (session_id, role, content, int(time.time()), tool_call_id, tool_name)
    )
    
    # 更新会话时间
    conn.execute(
        "UPDATE sessions SET updated_at = ? WHERE id = ?",
        (int(time.time()), session_id)
    )
    
    conn.commit()
    conn.close()
```

### 5.3 加载会话历史

```python
def load_session_history(session_id, limit=50):
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.execute(
        "SELECT role, content, tool_call_id, tool_name, reasoning "
        "FROM messages "
        "WHERE session_id = ? "
        "ORDER BY timestamp ASC "
        "LIMIT ?",
        (session_id, limit)
    )
    messages = [
        {
            "role": row[0],
            "content": row[1],
            "tool_call_id": row[2],
            "tool_name": row[3],
            "reasoning": row[4]
        }
        for row in cursor.fetchall()
    ]
    conn.close()
    return messages
```

### 5.4 搜索消息

```python
def search_messages(query, session_id=None):
    conn = sqlite3.connect(DB_PATH)
    
    if session_id:
        cursor = conn.execute(
            """
            SELECT m.session_id, m.role, m.content, m.timestamp
            FROM messages_fts f
            JOIN messages m ON f.rowid = m.id
            WHERE f.content MATCH ? AND m.session_id = ?
            ORDER BY m.timestamp DESC
            """,
            (query, session_id)
        )
    else:
        cursor = conn.execute(
            """
            SELECT m.session_id, m.role, m.content, m.timestamp
            FROM messages_fts f
            JOIN messages m ON f.rowid = m.id
            WHERE f.content MATCH ?
            ORDER BY m.timestamp DESC
            """,
            (query,)
        )
    
    results = [
        {
            "session_id": row[0],
            "role": row[1],
            "content": row[2],
            "timestamp": row[3]
        }
        for row in cursor.fetchall()
    ]
    conn.close()
    return results
```

def migrate_database(old_path, new_path):
    """迁移数据库到新位置"""
    shutil.copy2(old_path, new_path)
    
    # 验证迁移
    conn = sqlite3.connect(new_path)
    cursor = conn.execute("SELECT COUNT(*) FROM sessions")
    count = cursor.fetchone()[0]
    conn.close()
    
    print(f"迁移完成：{count} 个会话")

## 7. 性能优化

### 7.1 连接池

```python
from contextlib import contextmanager
import sqlite3

@contextmanager
def get_db_connection():
    conn = sqlite3.connect(DB_PATH, timeout=30.0)
    conn.execute("PRAGMA journal_mode=WAL")  # 预写日志模式
    conn.execute("PRAGMA synchronous=NORMAL")
    try:
        yield conn
    finally:
        conn.close()
```

### 7.2 批量插入

```python
def add_messages_batch(messages):
    """批量添加消息"""
    with get_db_connection() as conn:
        conn.executemany(
            "INSERT INTO messages (session_id, role, content, timestamp) VALUES (?, ?, ?, ?)",
            [(m["session_id"], m["role"], m["content"], m["timestamp"]) for m in messages]
        )
        conn.commit()
```

## 8. 安全考虑

- **敏感数据加密**：API 密钥存储在 `.env` 文件中，不进入数据库
- **访问控制**：数据库文件权限设置为 `600`（仅所有者可读写）
- **SQL 注入防护**：所有查询使用参数化查询
- **数据清理**：定期清理过期会话（可配置保留天数）

```sql
-- 删除 90 天前的会话
DELETE FROM sessions 
WHERE updated_at < strftime('%s', 'now', '-90 days');
```
